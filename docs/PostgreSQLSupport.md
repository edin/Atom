# PostgreSQL Support Plan

[Atom Framework](Index.md)

Atom currently has concrete database support for:

- SQLite
- MySQL / MariaDB
- PostgreSQL

`DatabaseDriverFactory` recognizes PostgreSQL aliases (`pgsql`, `postgres`, `postgresql`) and creates `PostgresDriver`.

This page tracks the small pieces needed to add PostgreSQL support without guessing.

## Driver

Needed:

- `Atom\Database\Driver\PostgresDriver`
- DSN format:

```text
pgsql:dbname=...;host=...;port=...
```

Expected driver methods:

- `compiler()`
- `schemaCompiler()`
- `schemaInspector()`
- `migrationRepositoryDriver()`
- `lockManager()`
- `resetter()`

Then update `DatabaseDriverFactory` so `pgsql`, `postgres`, and `postgresql` create `PostgresDriver`.

## SQL Compiler

Needed:

- `Atom\Database\Sql\Compiler\PostgresCompiler`

Differences from the current MySQL compiler:

- identifiers use double quotes: `"users"`
- limit/offset syntax should be `LIMIT n OFFSET n`, not `LIMIT offset, limit`
- delete queries with `LIMIT` are not directly supported by PostgreSQL
- raw expression quoting should use double quotes

The shared SQL AST walking logic now lives in `AbstractSqlCompiler`.
Dialect hooks are protected so individual compilers can override:

- `quoteIdentifier`
- `quoteExpression`
- limit compilation
- delete limit compilation

That lets MySQL, SQLite, and PostgreSQL share the AST walking logic while overriding dialect details.

## Schema Compiler

Needed:

- `Atom\Database\Schema\Compiler\PostgresSchemaCompiler`

Likely mappings:

| Atom Type | PostgreSQL Type |
| --- | --- |
| string | `VARCHAR(n)` |
| text | `TEXT` |
| integer | `INTEGER` |
| bigInteger | `BIGINT` |
| float | `DOUBLE PRECISION` |
| decimal | `DECIMAL(p, s)` |
| boolean | `BOOLEAN` |
| date | `DATE` |
| dateTime | `TIMESTAMP` |
| timestamp | `TIMESTAMP` |
| json | `JSONB` |
| binary | `BYTEA` |
| uuid | `UUID` |

Auto-increment needs a decision:

- first version can use `GENERATED BY DEFAULT AS IDENTITY`
- older PostgreSQL support could use `SERIAL`

## Schema Inspector

Needed:

- `Atom\Database\Schema\Inspector\PostgresSchemaInspector`

Can use `information_schema`:

```sql
SELECT COUNT(*)
FROM information_schema.tables
WHERE table_schema = current_schema()
  AND table_name = :table
```

Columns:

```sql
SELECT column_name
FROM information_schema.columns
WHERE table_schema = current_schema()
  AND table_name = :table
ORDER BY ordinal_position
```

## Migration Repository

Needed:

- `Atom\Database\Migration\Driver\PostgresMigrationRepositoryDriver`

This should be very close to the SQLite driver:

- double-quote identifiers
- `CREATE TABLE IF NOT EXISTS`
- `TIMESTAMP NOT NULL` for `applied_at`

Table existence can use `information_schema.tables` and `current_schema()`.

## Locks

Needed:

- `Atom\Database\Lock\PostgresDatabaseLockManager`

Use PostgreSQL advisory locks:

```sql
SELECT pg_try_advisory_lock(...)
SELECT pg_advisory_unlock(...)
```

The main decision is how to convert a string lock name into a stable advisory lock key. A first version could use a hash split into integer values.

## Resetter

Needed:

- `Atom\Database\Schema\Reset\PostgresDatabaseResetter`

Possible implementation:

```sql
SELECT tablename
FROM pg_tables
WHERE schemaname = current_schema()
```

Then drop tables with:

```sql
DROP TABLE "table" CASCADE
```

## Tests

No external PostgreSQL server is required for the first implementation slice.

Unit tests cover:

- factory creates `PostgresDriver`
- driver exposes PostgreSQL-specific collaborators
- query compiler SQL strings
- schema compiler SQL strings
- migration repository SQL strings

Integration tests against a real PostgreSQL database can come later and should be optional.

## Suggested Order

1. Done: refactor SQL compiler dialect hooks into `AbstractSqlCompiler`.
2. Done: add `PostgresCompiler` with SQL string tests.
3. Done: add `PostgresSchemaCompiler` with schema string tests.
4. Done: add migration repository driver.
5. Done: add schema inspector and resetter.
6. Done: add advisory lock manager.
7. Done: add `PostgresDriver`.
8. Done: enable PostgreSQL in `DatabaseDriverFactory`.
9. Add optional real database integration tests.
